DocTop

Inherits from: Object

DocTop organizes Documents, providing a list of all open Documents and placing them in one of two "slots".

It does this by polling Document.allDocuments (in the manner of SkipJack).

DocTop is displayed as a window with a listview of all open Documents and a control for the default Server at the bottom.

Documents can be moved between slots using the horizontal arrow keys;  option-arrow gives the Document the focus.  

DocTop constrains the height of its Documents, so vertically resizing its window resizes all of its documents.

Resize or moving a Document frees it from DocTop, it can be recaptures using  the arrow keys in the DocTop window.

DocTop supports SwingOSC and SwingDocument as well.  Most of the command keys found in the OSX version are 

implemented by SwingDocument, with the exception of the enter key (implemented as shift-return), cmd-. (shift-delete).  

DocTop and SwingDocument can be used with sclang (instead of the Supercollider application) to provide an approximation

of the standard OSX look and feel.  (At the moment there are no pull-down menus, eventually this will be implemented either 

within DocTop or as Swing menus.)

This is currently under development and is in a pre-alpha state.

Creation / Class Methods

*start(restore)

Starts DocTop using the current Document implementation

restore - if true, window positions are looked up from the archive.  This is implemented but has not been tested recently.


DocTop.start;

*swing

Starts SwingOSC, makes SwingDocument the Document implementation and starts DocTop.

To run within sclang, open a terminal window, move to the SC directory and type:

./sclang

DocTop.swing^[ // ctrl-[ is needed to get line interpreted

*stop

Stops DocTop.  Restores CocoaDcument if available.
