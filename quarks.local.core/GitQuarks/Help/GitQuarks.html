GitQuarks


Inherits from: Quarks : Object

Purpose


Modify method of installing / uninstalling Quarks to make these compatible with git. 


When sending a git archive to a remote repository such as on http://github.com/, symbolic links are translated to the original paths of the linked items. This means that when getting code from a remote git repository, those links will be broken. In order to preserve the state of installed quarks in a git repository, GitQuarks copies the quark directories into Extensions instead of linking them. 


Usage


Create an instance of GitQuarks instead of Quarks, giving it a local path, and then create a gui from it. 

See menu item created by Code class: 


CocoaMenuItem.addToMenu("Code", "Configure core quarks", nil, {

GitQuarks(localPath: Platform.userAppSupportDir +/+ "quarks.local.core").gui;

}),
