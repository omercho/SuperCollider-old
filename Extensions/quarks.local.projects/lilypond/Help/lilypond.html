Lilypond write (and play) scores in LilyPond format


v 0.01 - draft only 



LilyPond: open source music score software. See: http://lilypond.org/


Will be build bottom up, by generating scores, starting from simple ones and adding features. 


Class Lilypond is used to write the score.


Lilypond: Output a score on file in lilypond format. Variables: 

path: the path of the file

source: An object that creates a stream which generates the events of the score.

header: a dictionary of header elements and string values, used to create the header. 


file: the file where the score is written.



Some other thoughts


Define classes that represent basic notational elements, in a hierarchy. Use patterns to generate instances fo these in a sequence, which can then be translated into sound via SuperCollider and into a score in LilyPond format.  


Thougts for some basic element classes: 


ScoreEvent: Superclass for basic score event elements (note, chord, rest, maybe also tuple)

Holds possible common attributes of these elements: 

duration, dynamics, expression, grouping, fingering


Note: A note event. Parameters: pitch, duration, accidentals, 

Chord

Rest

Tuple: Sequence of notes connected with a beam.
